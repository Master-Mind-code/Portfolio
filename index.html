Étude de cas

Projet phare en profondeur

Comment j'ai conçu une plateforme bancaire conforme aux normes monétiques de l'UEMOA, de la commande de carte jusqu'à l'enrôlement biométrique.

Banque · Monétique CardFlow UEMOA · React + TypeScript

Gérer le cycle de vie complet d'une carte bancaire

Une banque ivoirienne avait besoin d'un outil unique pour orchestrer ses cartes de l'émission à l'enrôlement biométrique tout en restant strictement conforme aux normes GIM UEMOA et PCI-DSS.

Le défi

Des processus dispersés sur plusieurs systèmes, des contraintes réglementaires fortes (GIM UEMOA, PCI-DSS) et un besoin d'interface fiable pour des opérations bancaires sensibles, à fort volume.

Ma solution

  • 10 modules métiers unifiés : commande, personnalisation, enrôlement biométrique, installation d'applets, suivi du cycle de vie
  • Architecture React 18 + TypeScript, validation stricte avec Zod, état serveur géré via React Query
  • Intégration de 150+ endpoints API conformes GIM UEMOA, sécurisés par JWT
  • Design system interne de 50+ composants accessibles (Radix UI + TailwindCSS)

L'impact

Une plateforme unique, performante et maintenable, déployée en production avec une taille de bundle réduite de 81 % grâce à un travail ciblé d'optimisation (code-splitting, lazy loading, tree-shaking).
